
Q1: Why might we consider using Git?

Answer: Git is useful in allowing all teammates to contribute towards a 
project, while having safety features to ensure that nothing gets 
overwritten without approval and feedback from other teammates. This is 
crucial towards preventing further complications as the project complexity 
builds.

Q3: What is the difference between Merging and Rebasing?

Answer: Merging is combining the changes from one branch to another. It preserves the commit history between both branches. Rebasing is adding a branch to the tail of the other branch and then rewriting the commit history to make it seem 
sequential.
